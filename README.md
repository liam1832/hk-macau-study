## 四、如何维护图片

**换照片 / 二维码**：直接用同名文件覆盖 `assets/portrait.jpg`、`assets/wechat-qr.png` 即可。
（万一文件缺失，页面会自动回退到占位框，不会出现裂图。）

**增减 / 调整 Offer 分组**：Offer 墙按院校梯度分组，数据在 `app.js` 的 `OFFER_GROUPS`：
- 每条 `{ f:"offer-12", s:"香港中文大学", l:"硕士" }` 对应一张图（f=文件名、s=院校、l=学历）。
- 加图：放进 `assets/offers/`，在对应分组的 `items` 里加一行。
- 移动 / 改标签：直接改 `s`/`l`，或把整条挪到别的分组。
- 缺图的卡片会自动从墙上移除，不会出现裂图。

> Offer 图我已统一压到长边 ≤1100px，整组约 4–5MB，适合网页。原始大图在你的 Downloads。

## 六、设计规范

- 配色：深海军蓝 `#0D1B2A` · 香槟金 `#C8A96E` · 象牙白 `#F7F5F2` · 石灰灰 `#6B7280` · 炭黑 `#1F2937`（令牌都在 `styles.css` 的 `:root`）。
- 字体：思源宋体（标题）+ 思源黑体（正文）+ Cormorant Garamond（拉丁/数字）。
- 签名元素：金色「升学路径线」（Hero SVG + 案例卡迷你上扬线 + 字标）。
- 断点：≤980（案例/矩阵转两列）、≤860（导航收汉堡、上下排）、≤560（多数单列）、≤420（数据概览单列）。
- 无障碍：浅底金色/弱色文字已达 WCAG AA；键盘 focus 可见；支持 `prefers-reduced-motion`。

## 七、发布上线（GitHub Pages 一键脚本）

项目根目录有 `deploy.sh`。用你**独立的 GitHub 账号**发布：

```bash
gh auth login          # 选 GitHub.com，登录你新账号（多账号时用 gh auth switch 切换）
bash deploy.sh         # 默认仓库名 hk-macau-study；也可 bash deploy.sh 自定义名
```

完成后会打印固定网址：`https://<你的账号>.github.io/hk-macau-study/`（约 1 分钟生效）。
以后更新：改完代码 → 在 `site/` 里 `git add -A && git commit -m update && git push`。

> 注意：GitHub Pages 仓库是**公开**的，offer 图会公开可下载、可被搜索引擎收录。
> 已逐张检查并遮挡了两处露名的 offer（offer-06 / offer-10）；多张澳门 offer 带可扫描的验证二维码，如需更稳可在上线前覆盖。
> 其它托管（Vercel / Netlify / Cloudflare Pages）更适合以后绑定自定义域名，需各自账号登录。
